# 服务接入与域名绑定

## 一、新增业务服务

执行中文脚本创建服务模板：

```bash
bash scripts/create_service_compose.sh
```

脚本会在 `services/<服务名>/docker-compose.yml` 生成模板。

启动服务：

```bash
cd services/<服务名>
docker compose up -d
```

注意：模板默认只 `expose` 内部端口，不映射宿主机端口，因此多个 `8000` 可并存。

## 二、在管理面板绑定路由

打开管理面板后，填写：

1. 域名：例如 `api.example.com`
2. 服务名：例如 `new-api`
3. 服务端口：例如 `8000`
4. 备注：可选

点击“保存路由并重载 Nginx”后，配置会自动写入并重载。
如果勾选了 HTTPS，证书未申请前会先以 HTTP 工作，申请证书后自动切到 HTTPS。

## 三、绑定 Cloudflare DNS

有两种方式：

1. 面板里勾选“自动绑定 Cloudflare DNS”
2. 命令行手动执行：

```bash
python3 scripts/upsert_cf_dns.py --domain api.example.com
```

## 四、排错建议

1. 域名访问 502：
- 检查目标服务容器是否在 `proxy_net` 网络
- 检查服务名和端口是否与面板配置一致

2. 绑定 DNS 失败：
- 检查 `.env` 的 `CF_API_TOKEN`、`CF_ZONE_ID`
- 检查 token 权限是否包含 DNS 编辑

3. 路由修改后不生效：
- 在面板点击“手动重载 Nginx”
- 查看 `docker logs nginx-gateway`

4. HTTPS 未生效：
- 在面板点击“申请证书”
- 检查 `.env` 中 `CF_API_TOKEN`、`CF_ZONE_ID` 是否正确
- 确认 Cloudflare SSL/TLS 模式为 `Full (strict)`
