# 架构与能力说明

## 一、架构目标

本项目用于在全新 Ubuntu VPS 上快速搭建一个可维护的 Docker 反代平台，核心目标：

1. 多服务内部端口可重复（例如都为 `8000`），部署不冲突。
2. 通过域名将请求转发到不同容器。
3. 支持 Cloudflare DNS 绑定，操作尽量页面化、脚本化。
4. 支持自动申请 HTTPS 证书（Let's Encrypt + DNS-01）。

## 二、组件组成

1. `nginx-gateway`  
对外提供 `80/443` 入口，按域名反向代理到目标容器。

2. `proxy-panel`  
中文管理面板，能力包括：
- 增删路由
- 自动生成 Nginx 路由配置
- 自动触发 Nginx 重载
- 可选调用 Cloudflare API 绑定 DNS A 记录
- 页面展示环境检测结果

3. `proxy_net` Docker 网络  
所有业务容器接入该网络后，即可通过 `服务名:端口` 被 Nginx 访问，不需要映射宿主机端口。

4. `cert-agent`  
基于 `acme.sh` 通过 Cloudflare DNS API 自动申请证书，并写入 `certs/<域名>/`。

## 三、端口不冲突原理

不同服务即使都监听容器内部 `8000`，只要服务名不同，就可以共存：

- `api-a:8000`
- `api-b:8000`

Nginx 通过域名转发到不同服务名，因此无冲突。

## 四、建议实践

1. 业务容器尽量只 `expose` 内部端口，不要 `ports` 映射到宿主机。
2. 管理面板端口默认绑定 `127.0.0.1`，通过 SSH 隧道访问，避免公网暴露。
3. Cloudflare Token 建议最小权限（仅 DNS 编辑）。
4. Cloudflare SSL/TLS 模式建议设置为 `Full (strict)`，避免明文回源。
